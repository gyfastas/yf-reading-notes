VLM Agent Benchmark Survey · 2026

VLM Agent Bench
调研全景

VL关联性 · Docker要求 · 环境搭建 · 当前SOTA分数
本文档汇总 23 个主流 Agent / 视觉推理 Benchmark，重点关注三个维度： (1) 是否需要视觉/VL能力——GUI grounding、多模态搜索、Think-with-Image/Video 等 16 个 benchmark 要求 VLM，TAU2/BFCL/DeepPlanning 等 7 个纯文本可跑； (2) 环境复杂度——OSWorld/AndroidWorld/MobileWorld 需要 KVM 虚拟化或 Android 模拟器；MMSearch / ScreenSpot Pro / TIR-Bench / VideoHolmes 等最轻量（仅需 API key）； (3) Think with Image/Video——TIR-Bench（图像工��操作）、ViC-Bench（视觉中间状态生成）、VideoHolmes（跨帧线索搜索）、VideoThinkBench、TwiFF（未来帧预测推理）。 来源：Qwen3.5 技术报告 + VisualAgentBench (THUDM) + MMSearch/MMSearch-Plus + 独立调研（2026-03）。

02

VL 关联分析

判断标准：benchmark 的任务是否需要处理图像输入（截图、GUI 界面、网页视觉渲染）。

VL Required（视觉必须）
  • ScreenSpot Pro GUI grounding
  • AndroidWorld Android GUI
  • MobileWorld Android GUI
  • OSWorld Computer Use
  • VisualAgentBench 多环境VL
  • VisualWebArena 多模态Web
  • WebVoyager 多模态Web
  • ScreenAgent 桌面GUI
  • ScreenSuite 综合GUI
Text Only（纯文本可跑）
  • TAU2-Bench 对话客服
  • BFCL-V4 函数调用
  • DeepPlanning 长程规划
  • BrowseComp 文本为主†
  • WebArena 文本Web
  • WebChoreArena 文本Web
  • AgentBench 通用Agent
† BrowseComp 有 MM-BrowseComp 多模态扩展版（400题，需截图理解）
VLM 专属赛道：ScreenSpot Pro → OSWorld → AndroidWorld/MobileWorld 构成从"静态 GUI 定位"到"动态 OS 交互"的难度梯度。VisualAgentBench (THUDM) 覆盖最广（Embodied + GUI + CSS 设计），是最接近"全能视觉 agent"评测的 benchmark。如需快速测出 ThothVL 的 agent 基线，推荐优先跑 ScreenSpot Pro（轻量，无 Docker，纯 API）和 VisualWebArena（910题，Docker）。

Video + Gen No Docker
GitHub
LiuJunhua02/TwiFF
TwiFF-2.7M + TwiFF-Bench（评测集）

优先级建议（ThothVL agent 能力评测）：
① 最快上手（无 Docker）：ScreenSpot Pro（GUI grounding）+ MMSearch（多模态搜索）+ TIR-Bench（Think w/ Image 工具调用）+ WebVoyager（多模态 Web）
② 全面 GUI agent：AndroidWorld（Google，相对成熟）→ OSWorld（多 OS，AWS 可并行）
③ 最硬核 VL：VisualAgentBench（5 环境，含训练轨迹数据）+ MMSearch-Plus（带来源追溯的深度视觉搜索）
④ Think with Video：VideoHolmes（跨帧时序推理）+ TwiFF（2.7M 训练数据可直接用于 mid-training visual CoT）
⑤ Qwen3.5 报告对标（纯文本）：BFCL-V4 / TAU2-Bench / DeepPlanning / BrowseComp——反映底座 LLM agent 能力，不需要 VLM 特有能力。
