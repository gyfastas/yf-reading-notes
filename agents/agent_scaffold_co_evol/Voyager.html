Voyager

An Open-Ended Embodied Agent with Large Language Models
↗ arXiv:2305.16291 Wang, Xie, Jiang et al. · NeurIPS 2023
Voyager 是首个在 Minecraft 开放世界中展示持续自我进化能力的 LLM agent。它由三个核心组件驱动：(1) 自动课程（Automatic Curriculum）持续提议适当难度的探索目标；(2) 可增长的代码技能库（Skill Library）存储已验证的可执行技能，形成可复用的行为库；(3) 迭代式 prompting结合环境反馈和自我验证来生成可靠代码。Voyager 发现的独特物品数量是 baseline 的 3.3 倍，覆盖科技树比 ReAct/Reflexion 等方法快 15.3 倍。Skill Library 的持续增长本质上就是 scaffold 的共同进化。
01

三组件设计

Component 1
Automatic Curriculum
GPT-4 根据 agent 当前状态（物品栏、位置、已完成目标）自动提议下一个探索目标。目标按难度递进（先采集木头、再造工具、最后挖钻石）。类似 POET 的 minimal criteria：确保目标对当前能力适度挑战。
Component 2
Skill Library
每个成功完成的任务被封装为一个 JavaScript 函数（Minecraft Mineflayer API），存入向量数据库。函数包含代码 + 描述 + 前置条件。新任务时，通过 embedding 相似度检索最相关的已有 skill 作为参考。库持续增长 = scaffold 持续进化。
Component 3
Iterative Prompting
生成代码后在 Minecraft 中执行，获取环境反馈（执行错误、游戏状态变化）+ 自我验证（GPT-4 判断目标是否达成）。未通过则迭代修改代码。最多迭代 K 次，成功的代码才存入 Skill Library。
02

Skill Library 详解

Skill Library 是 Voyager 最重要的创新，也是"scaffold 共同进化"的最直观实例。

1
Skill 存储格式
每个 skill = (函数代码, 自然语言描述, embedding 向量)。代码是可直接执行的 Mineflayer JavaScript 函数，描述由 GPT-4 自动生成，embedding 用于检索时的相似度匹配。
2
Skill 检索
新任务到来时，将任务描述 embed 后在 Skill Library 中做 top-k 向量检索，找到最相关的已有 skill。这些 skill 的代码被注入 prompt 作为 few-shot 示例，大幅降低从零生成的难度。
3
Skill 组合与复用
新 skill 可以调用已有 skill 作为子程序（如 "craftDiamondPickaxe" 内部调用 "mineIronOre" 和 "smeltIron"）。这使 Skill Library 形成层次化结构，复杂行为由简单行为组合而成，类似软件工程中的函数抽象层次。
*
Skill Library 的开放式增长：与固定工具集不同，Voyager 的 Skill Library 没有上限。每完成一个新任务就可能新增一个 skill。随着 Library 增长，agent 能组合的行为空间指数扩大。这正是 scaffold co-evolution 的核心特征：scaffold（skill library）和 agent 能力（可解决的任务复杂度）互相推动增长。
03

开放式进化循环

三个组件形成正反馈循环：

Curriculum 提出新目标 → 检索相关 Skill 辅助代码生成 → 迭代验证后新 Skill 入库 → Library 增长使 agent 能力提升 → Curriculum 提出更难目标 → ...

+
实验数据：Voyager 在 160 次迭代中积累了约 70 个 skill，发现了 63 种独特物品（ReAct: 19, Reflexion: 21, AutoGPT: 24）。Skill Library 的增长曲线呈现明显的加速态势：后期 skill 复用率越来越高，新 skill 创建速度也越来越快，因为已有 skill 提供了更多组合基础。
04

对本研究的启发

Skill Library = Scaffold Library 的最佳实践模板
Voyager 的 Skill Library（代码 + 描述 + embedding + 层次化组合）为我们的 scaffold library 提供了完整的工程蓝图��Scaffold library 中每个条目应包含：scaffold 配置代码、自然语言描述、适用任务类型的 embedding、以及可组合的子 scaffold 引用。
Self-Verification 应内置于 scaffold 进化过程
Voyager 的自我验证（GPT-4 判断目标是否达成）确保只有 verified skill 才入库。同理，scaffold co-evolution 中新 scaffold 应通过自动验证（在验证集上测试 model 使用新 scaffold 的表现）才被接纳入 library，防止劣质 scaffold 污染库。
Curriculum + Skill Library 联动 = 难度自适应的完整方案
Voyager 证明了 curriculum（任务难度控制）+ skill library（scaffold 积累）的联动能实现开放式持续进步。在 midtrain 中，可以用类似架构：curriculum 控制训练任务难度递增，scaffold library 积累可复用的 tool/prompt 模板，两者共同驱动 model 能力的阶梯式提升。
代码作为 skill 的统一表示
Voyager 用 JavaScript 函数作为所有 skill 的统一表示，而不是自然语言描述或 JSON schema。这与 CodeAct 的理念一致，也印证了在 scaffold co-evolution 中应该以可执行代码作为 scaffold 的规范表示——代码既是执行载体，也是可检索、可组合、可验证的知识载体。
