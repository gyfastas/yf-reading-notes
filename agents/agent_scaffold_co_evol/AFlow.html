AFlow

Automating Agentic Workflow Generation
↗ arXiv:2410.10762 Zhang, Xiang et al. · ICLR 2025 Oral
AFlow 将 agentic workflow 的优化形式化为代码空间搜索问题：每个 workflow 是一段 Python 代码，其中 LLM 调用是节点，数据流是边。使用 Monte Carlo Tree Search (MCTS) 在代码修改空间中探索，结合经验回传（experience backpropagation）指导搜索方向。AFlow 在 6 个 benchmark 上以更低的 token 成本超越了人类手工设计的 workflow（如 CoT, Self-Refine, Role-play debate），是 ICLR 2025 的 Oral paper。
01

Workflow 建模

AFlow 的核心抽象：一个 agentic workflow = 一段 Python 程序，其中每个 LLM 调用是一个"operator node"（带有 prompt template 和输出 parser），节点之间的数据传递是"edge"。

Abstraction
Operator Nodes
预定义一组基本 operator：Generate（单次 LLM 调用）、Review（评审/打分）、Revise（基于反馈修改）、Ensemble（多路生成 + 投票）、Custom（自定义 Python 代码）。每个 node 包含 prompt template（可被搜索修改）和执行逻辑。
Abstraction
Workflow = Code
workflow 的拓扑结构、节点参数、数据流关系全部编码为 Python 代码。搜索 = 修改代码：添加/删除节点、改变连接方式、重写 prompt、调整循环次数。代码作为搜索空间的表示比 JSON/图结构更灵活（可以包含条件分支、循环、异常处理）。
02

MCTS 搜索策略

AFlow 使用 MCTS 在代码修改空间中探索最优 workflow。

S
Selection — UCB 选择节点
MCTS 树中每个节点是一个 workflow 代码版本。使用 UCB（Upper Confidence Bound）在探索（尝试未充分评估的 workflow）和利用（改进已知好的 workflow）之间平衡。
E
Expansion — LLM 生成代码修改
GPT-4 作为 optimizer，分析当前 workflow 代码和历史评测结果，生成代码修改建议（添加验证步骤、优化 prompt、调整流程结构等）。修改后的代码成为新的 MCTS 子节点。
V
Evaluation — Benchmark 评测
新 workflow 在 benchmark 子集上运行，获取 accuracy + token cost 双指标。Pareto 最优（高准确 + 低成本）被优先考虑。
B
Backpropagation — 经验回传
评测结果沿 MCTS 路径回传更新各节点的价值估计。成功的修改策略被强化（UCB 值上升），失败的被抑制。这使 optimizer 逐渐学会哪类修改更有效。
*
与 ADAS 的差异：ADAS 用 archive + greedy sampling（选最优起点），AFlow 用 MCTS（显式探索-利用平衡）。MCTS 的优势是有理论保证的探索效率，且 backpropagation 机制能传播远距离的搜索信号。但 MCTS 需要更多评估次数才能收敛。

实验结果

在 HumanEval、MBPP、MATH、GSM8K、HotpotQA、DROP 六个基准上，AFlow 发现的 workflow 在 5 个上超越了 CoT、Self-Refine、Debate 等人类设计方案，同时 token 消耗降低约 37%——更好且更便宜。

+
特别值得注意：AFlow 用 GPT-3.5 + 优化后的 workflow 达到的性能，在某些 benchmark 上接近 GPT-4 + 简单 CoT。这意味着 workflow 优化可以部分替代 model 能力提升——更好的 scaffold 让弱模型达到强模型的效果。
04

对本研究的启发

MCTS 是 scaffold 进化搜索的强 baseline
AFlow 证明 MCTS 在代码空间搜索中的有效性。在 scaffold co-evolution 中，���以用 MCTS 搜索最优 scaffold 配置：每个 MCTS 节点是一个 scaffold 版本，evaluation 是 model 使用该 scaffold 在验证集上的表现，backpropagation 更新 scaffold 变异策略的价值估计。
Token Cost 作为 scaffold 进化的优化目标之一
AFlow 同时优化 accuracy 和 token cost，发现两者经常可以同时改善。在 scaffold co-evolution 中，scaffold 的效率（model 使用该 scaffold 解决任务的 token 开销）也应作为优化目标，而不仅仅是任务成功率。高效 scaffold + 弱 model 可能优于低效 scaffold + 强 model。
Workflow 优化补偿 model 能力不足
AFlow 的 GPT-3.5 + 优化 workflow 接近 GPT-4 + CoT 的发现，强烈暗示 scaffold 优化和 model 训练是两个可以互补的维度。在 midtrain 阶段，如果 model 能力在某类任务上不足，先优化 scaffold（而不是增加训练数据/算力）可能是更高效的突破路径。
