Paired Open-Ended Trailblazer

POET: Endlessly Generating Increasingly Complex and Diverse Learning Environments and Their Solutions
↗ arXiv:1901.01753 Wang, Lehman, Clune, Stanley · GECCO 2019
POET 首次提出同时维护两个共同进化的种群——环境种群（environment population）和策略种群（agent policy population）——它们在开放式过程中相互推动对方变得越来越复杂。环境不断变异产生新挑战，agent 在不同环境间迁移（transfer），形成"stepping stones"效应：在一个简单环境中学到的能力可能恰好成为攻克另一个困难环境的跳板。该框架在 2D 双足行走任务上展示出远超单一课程学习和随机生成的性能。
01

核心机制

Dual Population
双种群共同进化
维护 N 个 (environment, agent) 对，每个 agent 专注于自己配对的环境。环境通过变异（mutation）逐渐变难/变化，agent 通过强化学习优化策略。两个种群在同一个循环中迭代进化——环境的变化驱动 agent 学习，agent 的能力水平决定哪些环境变异是"适度困难"的。
Minimal Criteria
最低准入标准
新生成的环境必须满足 minimal criteria：至少有一个现存 agent 能在其中获得正向 reward（不太难），但也不能让所有 agent 都轻松通过（不太简单）。这确保环境种群始终处于"适度困难"的前沿，避免产出无法学习的死亡环境或毫无挑战的退化环境。
Cross-Environment Transfer
跨环境迁移
定期尝试将一个环境中训练的 agent 策略 迁移到另一个环境。如果迁移后的策略在目标环境上比该环境原有 agent 表现更好，则替换。这一机制是"stepping stone"效应的核心——在环境 A 中获得的技能可能恰好是环境 B 的关键突破口。
Stepping Stone
踏脚石效应
多个看似"不相关"的简单环境各自培养的 agent 能力，组合后可能解决一个单独学习无法攻克的复杂环境。例如：agent 在平坦地面上学会稳定行走，在低矮障碍上学会跨步，这两项能力迁移后可以攻克高障碍+不平地面的组合挑战。
02

算法流程

POET 的主循环包含三个交替执行的阶段，每隔固定迭代数执行一次环境生成和迁移。

POET Main Loop:
Environment Population ← mutation → New Environments
↓ minimal criteria filter
Agent Population ← RL optimize on paired env
↓ periodic transfer attempts
Cross-Transfer → replace if better fitness
↓ repeat
1
环境变异与筛选
对现有环境进行随机变异（改变地形参数：间隙宽度、障碍高度、粗糙度等），生成候选新环境。用 minimal criteria 过滤：保留那些至少有一个 agent 能取得正向 reward、但没有任何 agent 已经"精通"的环境。通过筛选的环境加入种群，配对一个从最相关 agent 复制的策略。
2
Agent 在各自环境中训练
每个 (environment, agent) 对独立做若干步强化学习（使用 ES/进化策略）。Agent 只能看到自己配对的环境，不断优化在该环境上的表现。这一阶段是标准的 RL 训练，没有跨环境信息交互。
3
跨环境迁移尝试
每隔 T 轮，评估所有 agent 在所有环境上的表现。如果某个 agent a_i（来自环境 e_i）在环境 e_j 上的表现超过了 e_j 原有的 agent a_j，则用 a_i 替换 a_j。这种"全对全评估 + 最优替换"机制实现了 stepping stone 效应。
i
环境编码：在 2D BipedalWalker 实验中，环境用一个低维参数向量表示（stump height、gap width、roughness 等），变异就是对这些参数施加高斯噪声。环境复杂度的"开放性"来自参数空间的组合爆炸和不断推进的 minimal criteria 边界。
03

对本研究的启发

双种群共进化 = scaffold-model 共进化的原型
POET 的 (environment, agent) 双种群共进化与我们提出的 scaffold-model co-evolution 高度同构：scaffold（prompt/tool 设计）对应 environment，model weights 对应 agent policy。环境变得更具挑战性 = scaffold 要求提高；agent 变强 = model 能力提升。POET 证明了这种双向驱动可以开放式地持续提升，不陷入收敛。
Minimal Criteria 机制直接对应 curriculum 难度控制
POET 的 minimal criteria（不太简单不太难）与训练数据 curriculum 中的 difficulty-centric filtering 完全对应。在 scaffold co-evolution 中，可以用类似机制筛选训练任务：只保留当前 model 有一定概率解决但又非轻松通过的任务，实现自适应难度递增。
跨环境迁移 = 跨 scaffold 知识迁移
POET 的 cross-transfer 机制启示：在��个 scaffold 配置上训练的 model，应该定期尝试跨 scaffold 评估和策略迁移。一个 scaffold 上学到的能力（如某种 tool use pattern）可能成为攻克另一个 scaffold 挑战的关键跳板。这对 multi-scaffold 训练策略的设计有直接指导意义。
开放式进化需要多样性维护
POET 的一个教训：如果不维护环境种群的多样性，系统会快速收敛到少数类似环境。同理，scaffold co-evolution 需要显式维护 scaffold 多样性（不同 tool 组合、不同 prompt 结构、不同任务类型），避免 model 过拟合到单一 scaffold 模式。
