Large Language Models as Tool Makers

LATM: Amortizing Tool Creation Cost via Lightweight Tool Users
↗ arXiv:2305.17126 Cai et al. · Google DeepMind · ICLR 2024
LATM 提出两阶段的 tool making 范式：Stage 1 (Tool Making)——使用强模型（如 GPT-4）根据示例任务编写可复用的 Python 工具函数；Stage 2 (Tool Using)——弱模型（如 GPT-3.5）直接调用已创建的工具来解决新任务。关键洞察：工具创建是一次性高成本投资，而工具使用是低成本的重复操作。通过 tool caching，创建成本被摊销到大量后续任务调用中，在保持性能的同时大幅降低 API 开销。
01

两阶段设计

S1
Tool Making（强模型）
给定若干示例任务，GPT-4（tool maker）分析任务模式，编写一个通用 Python 函数作为 tool。函数需有清晰的 docstring、类型注解、错误处理。Maker 还会用示例验证工具正确性，必要时迭代修正。高 API 成本，但只执行一次。
S2
Tool Using（弱模型）
GPT-3.5（tool user）接收新任务 + tool 函数签名/docstring，决定如何调用工具解决问题。User 不需要理解 tool 内部实现，只需理解接口。低 API 成本，可复用于所有同类任务。
+
Dispatcher 机制：在 S1 和 S2 之间有一个 dispatcher，负责判断新任务是否能复用已有 tool（通过任务类型匹配），还是需要触发新一轮 tool making。Dispatcher 本身可以用弱模型实现，开销极低。
02

Tool 缓存策略

LATM 的经济性来自 tool caching：每个 tool 创建一次，存入 tool cache（一个 key-value 存储，key = 任务类型描述，value = Python 函数代码 + docstring）。后续同类任务直接从 cache 取出 tool 交给弱模型使用，无需重新调用强模型。

成本分析
摊销效果
假设 tool making 用 GPT-4 花费 $C_m$，tool using 用 GPT-3.5 花费 $C_u$。对 N 个同类任务：LATM = C_m + N * C_u vs 直接全用 GPT-4 = N * C_m。当 N 足够大时（实验中 N>3 即回本），LATM 成本远低于全用强模型。
性能保持
弱模型 + Tool 接近强模型
在 BigBench-Hard 等基准上，GPT-3.5 + LATM tool 的性能接近甚至达到 GPT-4 直接推理。Tool 把困难逻辑封装为函数，弱模型只需做简单的参数填充和结果解读，降低了对推理能力的要求。

# Stage 1: Tool Maker (GPT-4)
def make_tool(examples: List[Task]) -> Tool:
    prompt = TOOL_MAKER_PROMPT + format_examples(examples)
    code = gpt4(prompt)            # 生成 Python 函数
    for ex in examples:
        result = exec_tool(code, ex.input)
        if result != ex.expected:
            code = gpt4(REPAIR_PROMPT + code + error)
    return Tool(code=code, signature=extract_sig(code))

# Stage 2: Tool User (GPT-3.5)
def use_tool(task: Task, tool: Tool) -> Answer:
    prompt = TOOL_USER_PROMPT + tool.signature + task.input
    call_args = gpt35(prompt)      # 生成函数调用参数
    return exec_tool(tool.code, call_args)
04

对本研究的启发

Scaffold 进化 = Tool Making 的自动化版本
LATM 的 tool making 本质是 scaffold 的一次性构建：强模型生成可复用工具（scaffold 组件），弱模型在其上执行任务。我们的 scaffold co-evolution 将其推广为持续进化：不是一次创建 tool 就结束，而是随着 model 能力提升，不断更新和扩展 tool 库，两者协同改进。
Maker-User 分离 = 训练阶段的强弱模型分工
LATM 的 maker/user 分离启示：在训练数据生成阶段，可以用强模型（GPT-4/Claude）设计 scaffold（tool + prompt），弱模型（ThothVL）在 scaffold 上采集轨迹。Scaffold 质量由强模型保证，轨迹数据用于训练弱模型。这种分工与 LATM 的成本摊销逻辑一致。
Tool Cache = Scaffold Library 的实现参考
LATM 的 tool cache（key-value 存储，按任务类型检索工具）为我们的 scaffold library 提供了直接的实现参考：以任务类型为 key，存储经过验证的 scaffold 配置（prompt + tool set + workflow），新任务来时先检索 cache 再决定是否需要进化出新 scaffold。
