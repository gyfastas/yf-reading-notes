# Agent评估环境与评估集推荐指南

**基于 AgentGym 论文 (arXiv:2406.04151) 的实践指南**

---

## 1. 概述

AgentGym 提供了一个包含 **14个环境**、**89个任务类型** 的综合评估平台。本指南基于论文中的实验设计，为不同研究目标提供环境选择和评估集构建的推荐。

## 3. 按研究目标推荐环境组合

### 3.1 目标：评估通用Agent能力

**推荐环境组合**：WebShop + ALFWorld + SciWorld + Weather + BIRD-SQL

| 环境 | 数量 | 目的 |
|------|------|------|
| WebShop | 200 | 测试网页交互和信息检索 |
| ALFWorld | 200 | 测试常识推理和家务任务 |
| SciWorld | 200 | 测试科学推理和实验操作 |
| Weather | 20 | 测试基础工具使用 |
| BIRD-SQL | 200 | 测试代码生成能力 |
| **总计** | **820** | 覆盖5大核心能力维度 |

**评估重点**：
- 跨环境泛化能力
- 不同交互模式适应性（网页、文本、工具、代码）
- 平均成功率、任务完成效率

### 3.2 目标：评估网页Agent

**推荐环境组合**：WebShop + WebArena

| 环境 | 评估实例 | 最大轮数 | 特点 |
|------|----------|----------|------|
| WebShop | 200 | 10 | 电商场景，结构化搜索 |
| WebArena | 20 | 25 | 真实网站，复杂操作 |

**评估重点**：
- WebShop：商品搜索、属性匹配、价格筛选
- WebArena：信息检索、跨站点导航、内容管理
- 成功率、完成轮数、错误恢复能力

### 3.3 目标：评估具身Agent

**推荐环境组合**：ALFWorld + SciWorld + BabyAI

| 环境 | 任务类型 | 评估实例 | 特点 |
|------|----------|----------|------|
| ALFWorld | 6 | 200 | 家务任务，常识推理 |
| SciWorld | 30 | 200 | 科学实验，专业推理 |
| BabyAI | 40 | 90 | 网格世界，指令跟随 |

**评估重点**：
- 长程规划能力（ALFWorld平均13.3轮，SciWorld平均19.9轮）
- 环境交互效率
- 任务完成率与交互轮数的平衡

### 3.4 目标：评估工具使用能力

**推荐环境组合**：Weather + Movie + TODOList + Sheet + Academia

| 环境 | 动作数 | 评估实例 | 测试能力 |
|------|--------|----------|----------|
| Weather | 18 | 20 | 参数化API调用 |
| Movie | 16 | 20 | 多条件查询 |
| TODOList | 11 | 20 | 状态管理、增删改查 |
| Sheet | 20 | 20 | 复杂数据操作 |

**评估重点**：
- 工具选择正确性

### 3.5 目标：快速原型验证

**推荐环境组合**：WebShop + ALFWorld + Weather

**理由**：
- 覆盖网页、具身、工具三类场景
- 评估实例总数：200 + 200 + 20 = 420
- 可在较短时间内完成评估
- 环境相对稳定，易于调试

## 5. 评估流程建议

### 5.1 基准测试流程

```
Phase 1: 行为克隆基线 (Behavioral Cloning)
├── 使用 AgentTraj (6,130条轨迹) 训练基础Agent
├── 评估基础能力水平
└── 确定性能下界

Phase 2: 进化/RL训练 (Optional)
├── 使用 AgentEvol 方法进行多轮进化
├── 或使用 PPO/GRPO 进行RL训练
└── 每轮评估性能提升

Phase 3: 综合评估
├── 单任务评估：在单个环境上测试专业性
├── 多任务评估：在混合环境上测试通用性
└── 对比分析：与SOTA模型对比
```

### 5.2 实验设置建议

基于论文中的设置：

| 参数 | 建议值 | 说明 |
|------|--------|------|
| Temperature (评估) | 0 | 确定性输出 |
| Temperature (探索) | 0.7 | 增加探索多样性 |
| 学习率 (BC) | 1e-5 | 行为克隆阶段 |
| 学习率 (RL) | 1e-6 | RL/DPO阶段更稳定 |
| Epochs (BC) | 3 | 完整轨迹集 |
| Epochs (Evol) | 1/iteration | 每轮进化 |
| 最大轮数 | 见环境配置 | 不同环境不同 |

## 7. 实用建议

### 7.1 环境选择检查清单

- [ ] **研究目标明确**：是测试通用能力还是特定能力？
- [ ] **计算资源评估**：长程环境（如SciWorld）需要更多计算资源
- [ ] **评估时间预算**：完整AgentEval（1160实例）需要约1小时测试时间
- [ ] **基线对比需求**：是否需要与AgentEvol等方法对比？
- [ ] **数据需求**：是否需要使用AgentTraj进行预训练？

### 7.2 常见误区避免

1. **不要只选简单环境**：MAZE和Wordle虽然简单，但无法代表Agent的通用能力
2. **不要忽略工具环境**：Weather、Movie等工具环境是测试Agent实用能力的重要组成
3. **注意环境依赖性**：部分环境需要外部API（Weather、Movie），确保网络连接
4. **考虑评估成本**：WebArena使用真实网站，可能需要更多调试时间

### 7.3 扩展建议

如需开发新环境，建议遵循AgentGym的标准接口：

```python
# 标准接口设计
/createEnv     # 创建环境
/step          # 执行动作
/observation   # 获取观察
/reset         # 重置环境
```

并参考现有环境的难度设计：
- 简单环境：1-5轮交互
- 中等环境：5-15轮交互
- 复杂环境：15-30轮交互
