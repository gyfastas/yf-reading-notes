Kimi-K2 数据增强策略 (Data Rephrasing)：
论文明确描述了两种重phrase策略来提升 token utility：
  1. Knowledge Data Rephrasing：使用 LLM 以不同风格和视角重写文章，增加语言多样性同时保持事实完整性
  2. Mathematics Data Rephrasing：将高质量数学文档改写成 "learning-note" 风格，并翻译多语言数学材料
这种 "合成数据增强" 策略是 Kimi-K2 在 15.5T 数据上实现高效率的关键。

🔴 Kimi-K2: Data Rephrasing + Token Efficiency 点击展开

整体架构

Kimi-K2 的数据处理管线遵循 Kimi-K1.5 的方法论，但在数据增强方面引入了创新。核心目标是最大化 token efficiency（每 token 的性能提升）。

四域数据划分：
  • Web Text: 网页文本数据，主要知识来源
  • Code: 代码数据，用于编程能力
  • Mathematics: 数学文档，专门增强推理
  • Knowledge: 知识密集型文本

核心创新：Data Rephrasing Pipeline

1. Knowledge Data Rephrasing

针对知识密集型文本的困境：

  • 单 epoch 不足以吸收知识
  • 多 epoch 重复导致过拟合和收益递减

解决方案：

  1. Style- and perspective-diverse prompting: 受 WRAP 启发，使用精心设计的 prompt 在保持事实完整性的同时增强语言多样性
  2. Chunk-wise autoregressive generation:
    • 将长文档分割为片段
    • 逐个片段重 phrase
    • 重新拼接成完整段落
    • 避免 LLM 输出长度限制导致的信息丢失
2. Mathematics Data Rephrasing
  • 将高质量数学文档改写成 "learning-note" 风格（参考 SwallowMath）
  • 将其他语言的高质量数学材料翻译成英语
  • 增加数据多样性

质量验证

对每个领域执行：

  • Rigorous correctness validation: 正确性验证
  • Quality validation: 质量验证
  • Targeted data experiments: 针对性数据实验，确保高多样性和有效性
关键洞察：与简单重复不同，rephrasing 通过增加语言多样性而非重复来扩展高质量 token，避免过拟合。
🔴 Kimi-K2.5: Native Multimodal Joint Pre-training 点击展开

核心设计问题

给定固定的视觉-文本 token 预算，最优的视觉-文本联合训练策略是什么？

挑战传统观点：

传统方法（如 LLaVA、Qwen-VL）建议在训练后期以高比例（如 50%+）引入视觉 token，将多模态能力作为语言能力的事后补充。

Kimi-K2.5 发现：

  • 视觉比例对最终多模态性能影响很小
  • 早期融合 + 低视觉比例 > 晚期融合 + 高视觉比例

三阶段预训练流程

阶段数据说明
1. ViT Training 视觉数据 训练 MoonViT-3D 视觉编码器
2. Joint PT ~15T MM tokens 全程恒定比例混合文本和视觉 token
3. Mid-training 长文本数据 YaKN 渐进扩展至 256K 上下文

MoonViT-3D 设计

  • 初始化：SigLIP-SO-400M
  • NaViT packing：单图分块、展平、顺序拼接成 1D 序列，支持变分辨率
  • 3D 扩展：最多 4 个连续帧视为时空体，2D patch 联合展平打包
  • 时序池化：轻量级时序池化聚合每个时序块内的 patch，实现 4× 时序压缩
关键优势：
  • 图像和视频编码器完全共享参数
  • 统一嵌入空间
  • 图像预训练知识完全迁移到视频
  • 无需专门的视频模块或架构分叉

损失函数设计

Square-root-normalized per-token loss：

  • 从 per-sample loss 演进而来
  • 更好地平衡文本和多模态数据在训练中的贡献
  • 避免多模态数据淹没文本信号

多模态编码

  • Vision Encoder: SigLIP-2 (SO-400M 或 Large 300M)
  • Dynamic Resolution: 2D-RoPE + 基于输入大小的绝对位置嵌入插值
  • Merger: 两层 MLP 将 2×2 视觉特征压缩为单个视觉 token
  • Video: 显式文本时间戳 token 标记帧组

🩵 Intern-S1-Pro: PDF-based Scientific Data Pipeline 点击展开

核心设计动机

科学图像与自然图像的分布显著不同，需要更高的内容理解精度和细节关注。网页来源的科学图像在规模和领域密度上都不足，且原始 caption 通常简短且缺乏对齐。

PDF 作为理想数据源：
  • 包含高信息密度的图形（实验结果、统计图、结构图、公式推导）
  • 系统化、领域密集的专业视觉内容
  • 自然的跨模态对齐数据来源

Caption Pipeline 全流程

PDF → MinerU 2.5 提取 → pHash 去重 → 分类路由 → Caption 生成 → 0.5B 质量过滤 → 270B 输出

详细步骤

  1. PDF 提取 (MinerU 2.5)
    • 布局分析与结构识别
    • 检测并定位图表、公式、表格
    • 裁剪为标准化的子图像样本
  2. 内容去重 (pHash)
    • 感知哈希 (perceptual hashing)
    • 消除大规模冗余视觉内容
    • 处理科学文献中常见的重复图表
  3. 主题分类与模型路由
    数据类型Caption 模型特点
    科学图像InternVL3.5-241B专业领域特定描述
    非科学图像CapRL-32B通用密集 caption
  4. Caption 生成
    • CapRL (Captioning Reinforcement Learning): 基于 Qwen 2.5 VL 32B
    • 使用 RLVR (Reinforcement Learning with Verifiable Rewards)
    • 刺激密集图像 caption 能力
    • 多模板随机提示策略增强语言多样性
  5. 质量过滤 (0.5B 文本质量判别器)
    • 过滤乱码文本
    • 过滤重复表达
    • 过滤低信息密度内容

数据规模与覆盖

  • 总输出: 约 270B tokens 高质量科学图文 caption 数据
  • 覆盖领域: 生命科学、化学、地球科学、材料科学
  • CPT 总量: 6T tokens（含科学和通用数据）

科学数据冲突解决

三大策略：
  1. 结构化科学数据转换：
    • Template Construction: 异构输入-输出对转换为语法正确的叙述文本
    • Task Form Transformation: 数值符号映射到有科学意义的描述
  2. 科学数据多样化：
    • Prompt Diversification: 同一概念提供数十种不同指令
    • Rollout Mechanism: 生成完整推理链，将知识回忆转化为逻辑推理
  3. System Prompt Isolation：
    • 科学数据和通用数据注入互斥的系统级前缀
    • 创建独立的上下文处理环境

ViT 预训练三阶段

阶段方法训练样本Token 比例目标
1. MIM Masked Image Modeling + 2D RoPE 2.2B4.0%增强视觉几何和结构感知
2. Contrastive Native-Resolution Contrastive Learning 4.8B91.2%图文对齐

阶段 1: MIM with 2D RoPE

  • 教师模型: EVA02-CLIP-E
  • 学生模型: 随机初始化，使用 2D RoPE
  • 掩码比例: 75% 图像 patch 和对应 RoPE 嵌入
  • 目标: CLIP 特征
  • 损失: 余弦相似度损失
  • 关键发现: 2D RoPE 使学生具备强大的原生动态分辨率识别能力

阶段 2: Native-Resolution Contrastive Learning

  • 视觉编码器: MIM 训练的学生模型初始化
  • 文本编码器: EVA-02-CLIP-E 的文本编码器初始化
  • 特征聚合: attention pooling 聚合 patch 特征为 1280 维图像嵌入
  • 损失: SigLIP loss + SuperClass loss

阶段 3: Omni-modal Pre-training

  • 框架: MiCo
  • 数据: 视频帧、音频、视觉 caption、音频 caption 的对齐元组
  • 处理: ViT 编码视频帧和音频，独立文本编码器处理 caption
  • 目标: 嵌入对齐

多模态预训练数据类别

  1. Generic Image-Text Pairs & Knowledge: 通用图文对和知识数据
  2. OCR: 光学字符识别数据
  3. Visual Grounding & Counting: 视觉定位和计数
  4. 3D Spatial Understanding: 3D 空间理解
  5. Video: 视频理解
  6. STEM: 科学、技术、工程、数学
  7. GUI: 图形用户界面
设计原则：
  • 更好的训练效率：专门的 ViT 预训练避免联合训练的不稳定性
  • 全面的数据利用：充分利用所有模态数据
  • 架构一致性：ViT 预训练和 VLM 阶段架构保持一致

质量分层策略 (Appendix C.1.2):

  • Nemotron-CC 风格分层: 6个轻量级打分器/分类器集成，分为 High/Medium-High/Medium/Medium-Low/Low
  • 保留策略: 仅保留 High/Medium-High/Medium；书籍和论文在退火阶段仅保留 High/Medium-High
  • Embedding-based 聚类再平衡: 中英语料嵌入 → K-means (100k+ clusters) → 对过度代表的聚类下采样
  • 知识密集挖掘: 两阶段流水线 - (1) 高价值实体检索知识密集文档 (2) 受控改写和QA合成提升可学习性

训练课程进展:

  1. Stage 1: 4K上下文开放域预训练，最大化覆盖和基础能力
  2. Stage 2 (Annealing): 数据混合向代码和PR/Issue/Commit中心来源退火，增加高质量知识和推理密集样本比例
  3. Mid-training Stage 1: 32K特化，21%预训练数据replay防止分布偏移
  4. Mid-training Stage 2: 128K长上下文特化，合成长视野推理 + 自然长文档

来源: arXiv:2602.10604v1, Section 4.2.1 & Appendix C

关键洞察：
  • Qwen3-VL 采用4阶段渐进：从仅训练merger(67B)到全参数(1T)再到长上下文(256K)，每个阶段数据配比不同
  • GLM-5 三阶段长文本：32K(1T) → 128K(500B) → 200K(50B)，呈指数递减
  • Intern-S1-Pro 仅CPT：没有从头预训练，而是在基座模型上做6T continued pre-training
  • Step 3.5 Flash Agent导向：PR/Issue/Commit数据(100B+) + 工具使用合成数据，支持软件工程Agent工作流

Kimi-K2.5 "Native Multimodal" 发现：

  • 视觉比例对最终多模态性能影响很小
  • 早期融合 + 低视觉比例 > 晚期融合 + 高视觉比例

loss_weight = sqrt(token_count_ratio)

05.5 — DCLM 与数据筛选原理详解

DCLM (DataComp-LM) 简介

DCLM 是一个用于语言模型训练数据集控制的测试平台，目标是系统性地改进语言模型训练数据。它提供了一个标准化的 240T tokens 语料库（从 Common Crawl 提取），基于 OpenLM 框架的有效预训练配方，以及 53 个下游评估任务。

DCLM-7B 基准结果：
  • 使用 DCLM-BASELINE (2.6T tokens) 训练 7B 参数模型
  • MMLU 5-shot 准确率达到 64%
  • 相比 MAP-Neo（之前的开源数据 SOTA）提升 6.6个百分点
  • 相比 Llama 3 8B 使用 6.6× 更少计算量 达到相近性能

DCLM 数据管理流程

阶段 方法 说明
文本提取 Resiliparse + Trafilatura 从 Common Crawl WARC 文件提取干净文本
启发式清洗 RefinedWeb 过滤器 语言分类、长度过滤、URL 过滤等
去重 Bloom Filter / MinHash 移除精确或近似重复内容
质量筛选 Model-based filtering 使用 fastText bigram 分类器进行质量评分

核心发现：Model-based Filtering 是关键

关键实验结果 (arXiv:2406.11794 Sec 4.4)：

DCLM 对比了多种模型筛选方法：

  1. PageRank - 基于文档链接关系
  2. SemDedup - 语义去重，基于模型表示
  3. BGE 嵌入线性分类器 - 基于预训练 BGE 文本嵌入
  4. AskLLM - 提示 LM 判断文档是否有帮助
  5. Perplexity filtering - 保留低困惑度序列 (CCNet 风格)
  6. Top-k average logits - 平均 top-k 模型 logits

胜出者：简单的 fastText bigram classifier 配合精心选择的正负样本，在所有分类器中表现最佳。

意外发现：人工质量判断在识别高质量训练数据方面价值有限。

Fuzzy Deduplication 原理详解

网页爬取数据集通常包含大量重复或近似重复的数据字符串。去重有双重作用：(1) 通过减少记忆化提升性能；(2) 增加数据多样性。

📐 MinHash (局部敏感哈希) 点击展开

原理：MinHash 是一种局部敏感哈希 (Locality-Sensitive Hashing, LSH) 技术，用于基于 Jaccard 相似度对集合进行分组。

数学基础：
  • Jaccard 相似度：$J(A,B) = \frac{|A \cap B|}{|A \cup B|}$
  • MinHash 估计：$P[h_{min}(A) = h_{min}(B)] \approx J(A,B)$
  • 通过多组哈希函数的碰撞概率估计文档相似度

DCLM 实现细节：

  • n-gram 大小：5 tokens
  • 目标 Jaccard 相似度：0.8
  • 哈希排列数：1,395 个（分为 93 个 bucket，每 bucket 15 个 hash）
  • 优化：相比 Lee et al. 的 9,000 排列，DCLM 使用更高效的参数设置

Suffix Array 集成：

MinHash 与后缀数组结合使用，后缀数组首先识别和移除大型语料库中的子字符串重复，然后 MinHash 进行文档级去重。

优缺点：

优点缺点
可处理大规模数据集计算成本高（大量排列）
理论保证 (Jaccard)超大规模时扩展性受限
成熟的开源实现内存需求大
🔮 Bloom Filter (布隆过滤器) 点击展开

原理：Bloom filter 是一种空间高效的概率性数据结构，支持集合成员查询。

核心特性：
  • 无假阴性：如果元素在集合中，查询永远不会返回 False
  • 有假阳性：如果元素不在集合中，查询可能错误地返回 True
  • 空间复杂度：亚线性空间 $O(n)$，远低于存储所有元素

数学公式：

  • 最优哈希函数数：$k = -\frac{\ln \epsilon}{\ln 2}$，其中 $\epsilon$ 是期望的假阳性率
  • 最优位数组大小：通过求解 $\epsilon = (1 - e^{-kn/m})^k$ 得到 $m$

DCLM-Baseline 使用的方法：BFF (Big Friendly Filter)

基于 Soldaini et al. 的精确去重扩展，支持：

  • 文档级去重：整篇文档 vs 语料库
  • 段落级去重：检测文档内的重复段落
  • 同时处理：文档和段落去重同时进行
BFF 工作流程：
  1. 初始化：估计语料库 token 数量和期望假阳性率
  2. 计算最优哈希函数数 $k$ 和位数组大小 $m$
  3. 遍历每个文档的每个段落：
    • 计算哈希值
    • 查询 Bloom filter（如果存在则标记为重复）
    • 插入 Bloom filter
  4. 移除被标记为重复的段落或文档

优缺点：

优点缺点
极高的空间效率存在假阳性（可配置）
扩展性好（>10TB）无法删除元素（标准版）
比 MinHash 快得多只能判断存在性，不能计算相似度

DCLM 实验结果：

MinHash 和 Bloom Filter 在下游性能上表现相当（7B-2x 规模上 CORE 分数差距 < 0.2），但 Bloom Filter 在超过 10TB 数据集上扩展性更好。因此 DCLM-Baseline 使用 Bloom Filter，其他实验使用 MinHash。

⚖️ MinHash vs Bloom Filter 对比与模型应用 点击展开
维度 MinHash + Suffix Array Bloom Filter (BFF)
去重级别 文档 vs 文档 (pairwise) 文档 vs 语料库 (corpus-level)
相似度度量 Jaccard similarity (可配置阈值) 存在性检测（0/1）
计算成本 高（需要大量哈希计算） 低（几次哈希查找）
内存需求 高（存储签名） 中等（位数组）
扩展性 中等（>10TB 时困难） 优秀（轻松处理 >10TB）
适用场景 需要相似度分数的精细控制 大规模快速去重
DCLM 使用 实验和消融研究 DCLM-Baseline 生产版本

在 GLM-5 中的应用

GLM-5 在其数据管线中使用了基于 DCLM 的改进策略：

  • DCLM 分类器：基于 sentence embeddings 的额外分类器层
  • Fuzzy dedup：代码数据扩展 28% 的同时保持质量
  • World Knowledge 分类器：专门针对 Wikipedia 优化的变体

在其他模型中的应用

  • RefinedWeb：Bloom filter + 启发式过滤
  • RedPajama：MinHash + URL 去重
  • Dolma-V1：Suffix array + Bloom filter 组合
GLM-5 数据管线中的 DCLM 实践：
  • Web 数据：在 GLM-4.5 基础上引入基于 sentence embeddings 的 DCLM 分类器，识别并聚合超出标准分类器的高质量数据
  • 长尾知识：使用针对 Wikipedia 条目和 LLM 标注数据优化的 World Knowledge 分类器
  • 代码数据：修复 Software Heritage 元数据对齐问题，采用更精确的语言分类管线，模糊去重后获得 28% 的 token 增长
  • 低资源语言：为 Scala、Swift、Lua 等训练专门分类器
  • 数学科学：开发 chunk-and-aggregate 评分算法提高长文档评分准确性，严格避免使用合成/AI生成/模板数据

🧬 Intern-S1-Pro 科学数据冲突解决机制 点击展开

科学数据（实验观测、化学公式、结构化文献）与通用数据（新闻、社交媒体）直接混合会导致 "distribution shift" 和 "negative transfer"。Intern-S1-Pro 采用三大策略解决：

1. 结构化科学数据转换

  • Template Construction: 将异构输入-输出对转换为语法正确的叙述文本
  • Task Form Transformation: 对列表、矩阵等抽象输出，结合领域科学先验将数值符号映射到有科学意义的描述性答案

2. 科学数据多样化

  • Prompt Diversification: 对同一概念提供数十种不同指令，扩展模型泛化边界
  • Rollout Mechanism: 结合科学先验知识，利用强基座模型辅助生成完整推理链，将知识回忆转化为逻辑推理

3. System Prompt Isolation

在训练周期中注入互斥的系统级前缀：
  • 科学数据：特定系统提示
  • 通用数据：不同系统提示
为模型创建独立的上下文处理环境，有效减少数据冲突，提高训练稳定性和效果。
核心洞察：为什么 Intern-S1-Pro 的 Pipeline 是独特的？
  1. PDF-first 策略：与从网页抓取不同，直接从 PDF 文献提取保证专业性和信息密度
  2. 视觉感知去重：pHash 专门针对科学图像的视觉相似性，而非文本相似性
  3. 模型路由：根据内容类型动态选择不同 caption 模型，而非单一模型处理所有数据
  4. 领域专门化：270B 全部面向科学，而非通用数据中包含少量科学内容
  5. 系统隔离：独特的 System Prompt Isolation 解决数据冲突问题
