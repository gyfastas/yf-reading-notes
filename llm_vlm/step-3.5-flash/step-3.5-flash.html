01 — TL;DR

核心定位：Frontier 级推理 + Agent 能力的高效开源 MoE。196B 总参数但仅 11B 激活，保持大模型的"记忆容量"同时以小模型的速度推理。全栈开源（权重 + 训练框架 SteptronOss + 预训练 / Mid-training checkpoints）。
效率亮点：
  • 推理速度 100-300 tok/s（编码任务峰值 350 tok/s）
  • MTP-3 单次 forward pass 预测 4 tokens
  • 推理成本约为 DeepSeek-V3.2 的 1/6
  • AIME 2025: 97.3%, SWE-Bench: 74.4%

MoE 设计

Fine-Grained 路由：288 个 routed experts + 1 个 always-on shared expert。每 token Top-8 选择。

效果：保留 196B 模型的"知识容量"，但推理开销仅 ~11B 密集模型水平。

3:1 Sliding Window Attention

混合注意力：每 4 层中 3 层用 Sliding Window (window=512)，1 层用 Full Attention。

优势：大幅降低长上下文计算开销（256K context），同时通过 full attention 层保持全局信息聚合。

MTP-3 (Multi-Token Prediction)

MTP Head 结构：Sliding-window attention + Dense FFN，单次 forward pass 预测 4 个 tokens。

Head 参数：0.81B（独立于 backbone 的 196B）

速度收益：100-300 tok/s 常规使用，编码任务峰值 350 tok/s。

06 — 关键启发

  • Fine-grained MoE (288 experts)：比 DeepSeek-V3 的 256 更细粒度，配合 Top-8 实现极低激活比
  • 3:1 SWA 是长上下文的高效方案：不需要全部层做 full attention，混合策略大幅降成本
  • MTP 是推理加速的关键：4-token 预测 + speculative decoding 协同实现 3x+ 加速
  • Mid-training checkpoint 开源：社区可从 mid-train 阶段开始定制化训练
