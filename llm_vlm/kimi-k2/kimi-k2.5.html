Kimi K2.5: Visual Agentic Intelligence

Moonshot AI | 1T MoE (32B active) | ~15T Multimodal Tokens | MoonViT-3D | Agent Swarm

01 — TL;DR

核心定位：在 Kimi-K2 基座上，通过 ~15T 混合视觉-文本 tokens 的原生多模态联合预训练，构建视觉 + 代理一体化模型。不是"先训语言再加视觉"，而是联合训练同时增强两种模态。
三大创新：
  • MoonViT-3D：原生分辨率视觉编码器，图像/视频完全共享参数，4 帧时序压缩
  • Zero-Vision SFT：Post-training 阶段只用文本 SFT 即可激活视觉推理（因为联合预训练已建立对齐）
  • Agent Swarm：自导向并行代理编排，复杂任务延迟降低 4.5x

03 — MoonViT-3D 视觉编码器

核心设计：统一架构 + 完全共享参数 + 一致嵌入空间，同时处理图像和视频。

图像处理

  • 从 SigLIP-SO-400M 初始化
  • 采用 NaViT patch packing：将图像切成 patches → 展平 → 拼成 1D 序列
  • 原生分辨率输入，无需子图分割/拼接，支持高达 4K 分辨率

视频处理（3D 扩展）

  • 连续 4 帧为一组，通过共享 MoonViT 编码后在 patch 级别做时序平均
  • 等效 4x 压缩：相同上下文窗口内可处理 4 倍长的视频
  • 支持 2K 分辨率视频

继续预训练

训练目标：纯 cross-entropy loss（不含对比损失，区别于 Kimi-VL）

训练数据：图像-文本对 + 视频-文本对。文本端包含：alt text、合成 caption、grounding bbox、OCR text。

04 — 多阶段训练 Pipeline

三阶段，共 ~15T tokens：
  1. Standalone ViT Training：独立训练视觉编码器，建立鲁棒的原生分辨率特征
  2. Joint Pre-training：在 K2 checkpoint 基础上，视觉+语言联合训练，同时增强两种能力
  3. Mid-training：高质量数据精炼 + 长上下文激活（扩展到 256K）

联合预训练的关键洞察

颠覆传统认知：传统做法是在 LLM 训练后期才引入视觉 tokens（如 50%+ 比例），把多模态当作后置附加。

K2.5 发现：早期就开始联合训练 + 适当比例的视觉 tokens 效果更好——语言和视觉能力同时增强，而非此消彼长。

05 — Post-Training

Zero-Vision SFT

最反直觉的发现：Post-training 阶段只用纯文本 SFT，即可激活视觉推理和工具使用能力。

加入人工设计的视觉轨迹数据反而损伤泛化性。原因：联合预训练已经建立了足够强的视觉-文本对齐，文本 SFT 的能力提升自动迁移到视觉侧。

Joint Multimodal RL

打破模态边界：RL 的 domain experts 不按输入模态划分（不是"视觉专家"/"文本专家"），而是按能力维度划分：
  • Knowledge / Reasoning / Coding / Agentic / ...
  • 每个 domain expert 同时学习纯文本和多模态 queries
  • GRM（Generative Reward Model）同样跨模态优化
效果：文本端学到的能力提升自动泛化到视觉端，反之亦然。

QAT INT4 量化

训练阶段原生 INT4：不是后训练压缩，而是在训练过程中引入量化噪声。模型学到了对低精度鲁棒的表征。

效果：FP16 → INT4 推理速度 2x，精度无损。

06 — Agent Swarm

自导向并行代理编排框架：
  • 动态分解复杂任务为异构子问题
  • 并发执行多个 sub-agent
  • 延迟降低 4.5x（vs 单 agent baseline）

08 — 对 ThothVL 的启��

  • 联合预训练 >> 后置视觉：早期就混入视觉 tokens，语言和视觉能力同时增强
  • Zero-Vision SFT：如果预训练对齐够强，post-training 可以极大简化视觉数据需求
  • 3D ViT 时序压缩：4 帧平均是简单高效的视频压缩方案，值得在 ThothVL 上验证
  • 能力维度 > 模态维度：RL 按能力而非模态划分 domain，促进跨模态泛化
  • QAT INT4：训练阶段就引入量化，部署时直接 2x 加速
