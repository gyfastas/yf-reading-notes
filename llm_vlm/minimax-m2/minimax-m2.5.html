03 — Forge RL 框架（核心创新）

Agent-Native RL：不是在静态数据上优化，而是将模型部署到真实环境中——代码仓库、浏览器、办公应用、API 端点——基于任务完成结果做强化学习。

CISPO 延续 + Process Reward

新增：为解决 agent 长 rollout 中的 credit assignment 问题，引入 process reward 机制。在每个生成步骤监控质量，而非只看最终任务完成。

这与 2512.07783 (pre-train/mid-train/RL interplay) 的 Finding 4（process-level reward 抑制 reward hacking）高度一致。

涌现行为：Spec-Writing

训练涌现：M2.5 在真实环境 RL 训练中自发学会了"先写 spec 再写代码"——从软件架构师视角分解需求、规划结构、设计 UI，然后才动手实现。

解释：当 reward 信号是端到端任务完成时，模型发现预先规划可以减少后续错误、节省总 token 消耗。这是一种learned planning behavior。

08 — 关键启发

  • Forge RL "解耦层"：Agent scaffold 与训练引擎解耦，模型泛化到新 scaffold 而非过拟合特定工具接口
  • Process Reward for Agent：长 rollout 的 credit assignment 需要步骤级监控，与纯 outcome reward 互补
  • 涌现 Spec-Writing：端到端环境 RL 可以涌现出 planning behavior，不需要显式教
  • 230B/10B = Agent 最优点？三代模型固定此规格，说明对 agent 场景（高频多轮调用），10B active 可能是延迟/质量最优平衡点
  • 同架构，纯 RL 方法驱动 15 ppt SWE-Bench 提升：从 M2 到 M2.5，post-training 方法的迭代本身就能带来巨大增益
