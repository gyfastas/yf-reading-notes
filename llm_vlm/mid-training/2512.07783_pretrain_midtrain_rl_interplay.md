# On the Interplay of Pre-Training, Mid-Training, and RL on Reasoning Language Models

> **arXiv:** [2512.07783](https://arxiv.org/abs/2512.07783)
> **作者:** Charlie Zhang, Graham Neubig, Xiang Yue (CMU LTI)
> **提交时间:** 2025年12月
> **精读日期:** 2026-03-22

---

## TL;DR

用完全受控的合成实验框架（GSM-Infinite DAG），系统研究 pre-train → mid-train → RL 三阶段的相互作用，得到四条可操作结论：

1. **RL 只在"能力边界"处产生真实增益**（pass@128）
2. **Pre-training 需要 ≥1% 的覆盖率**，RL 才能泛化到新 context
3. **同等 compute 下，mid-training + RL 显著优于纯 RL**（+10.8%）
4. **Process-level reward 能有效抑制 reward hacking**

---

## 研究背景与动机

### 核心争议

文献中存在两个对立阵营：

| 阵营 | 代表工作 | 核心观点 |
|------|---------|---------|
| "RL 只是 refiner" | Yue et al. (2025), Wu et al. (2025), Zhao et al. (2025) | RL 不提升 pass@128，只是 sharpen 已有技能 |
| "RL 能扩展能力" | ProRL (Liu et al. 2025), Yuan et al. (2025), Sun et al. (2025) | RL 在 OOD 合成任务上有显著增益 |

本文的贡献：这两种说法**针对不同难度域**，并不矛盾。受控实验统一了这个争议。

## 实验框架

### 合成数据生成（GSM-Infinite）

- **DAG 结构**：节点 = 变量，边 = 算术依赖，复杂度 `op(G) = |E|`（边数 = 运算数）
- **Context 模板**：同一 DAG 可套入不同叙事（动物园、学校…），surface 变但 structure 不变
- **无污染拆分**：pre/mid/post-training 数据完全隔离

### 泛化维度

| 维度 | 定义 |
|------|------|
| **外推泛化（Depth）** | 测试题的 op 数 > 训练最大 op 数 |
| **上下文泛化（Breadth）** | 相同结构，但 context template 未见过 |

### 模型规格

- 100M 参数 Qwen2.5-style decoder-only，从零预训练
- 预训练规模：10B tokens（遵循 Chinchilla scaling）
- 自定义 BPE tokenizer，vocab 2200，max seq len 2048

### RL 算法

GRPO（Group Relative Policy Optimization），4 种 RL 数据难度区间：
- ID（op=7–10）、Mixed（op=9–12）、OOD-edge（op=11–14）、OOD-hard（op=17–20）

### 评估协议

**Process-verified evaluation**：intermediate reasoning steps 和 final answer **同时正确**才算对。同时报告 pass@1 和 pass@128。

## 四条核心发现

### Finding 1：RL 真实增益的条件（Section 3）

**实验设置**：固定 pre-training（op=2–10），改变 RL 数据难度。

**结论：RL 要产生真实能力增益（pass@128↑），需满足两个条件：**

1. 任务未被 pre-training 充分覆盖（存在"headroom"）
2. RL 数据处于模型的**能力边界**（edge of competence）——太容易无益，太难也无益

**具体数字：**
- ID 任务（op=7–10）：RL 提升 pass@1，但 **pass@128 几乎不变** → 只是 sharpening
- OOD-edge 任务（op=11–14 RL 数据）：pass@128 提升 **+42%** → 真实能力增益
- OOD-hard 任务：用 op=11–14（而非 op=17–20）的 RL 数据效果最好

**统一了争议**：
- 先前发现"RL 没用"的工作用的是 well-covered domains（数学、代码），符合本文 ID 预测
- 先前发现"RL 有大收益"的工作用的是 OOD 合成任务，符合本文 OOD-edge 预测

**实践指导**：过滤 RL 数据集，保留 pass@1 ≈ 0 但 pass@k > 0 的样本（自然形成 self-paced curriculum）。

---

### Finding 2：Pre-training 覆盖率决定 RL 的泛化边界（Section 4）

**实验设置**：固定 RL recipe（50% context A + 50% context B），改变 pre-training 中 context B 的比例。

**结论：存在一个"最低覆盖率阈值"，约 ~1%。**

| Pre-training context B 比例 | RL 后 context B 表现 |
|----------------------------|---------------------|
| 0% 或 0.1% | RL **完全无法**迁移到 context B |
| ≥1% | RL 稳健泛化，pass@128 提升 **+60%** |

**Replication vs. Creation 分析：**
- 简单任务（op=2–10）：模型复用 context A 的 patterns
- 复杂任务（op=11–20）：有足够 context B 基础后，模型生成**全新结构**（真正的组合泛化）

**"≥1% 覆盖率"的含义**：context B 的 atomic-level 样本（最简单形式，op=2）在 **pre-training 语料中所占的比例**。
- 这里的 context B 是一套不同叙事模板（比如把"动物园"改成"学校"），结构完全相同但 surface form 不同
- RL 阶段不能凭空创造 pre-training 里完全没见过的知识，但只需"见过一点点"即可
- 对 VLM 的启发：想让 RL 泛化到某类视觉推理 pattern，该 pattern 的**基础形式**在 pre/mid-training 里至少要有 ~1% 出现

**实践指导**：宁可用 1% 的 atomic-level 广覆盖，不要堆复杂样本。**数据广度 > 数据复杂度**。

### Finding 3：Mid-training 显著提升固定 compute 下的性能（Section 5）

**Compute Budget 形式化：**
- Mid-training cost: `T_mid = supervised tokens processed`
- RL cost: `T_RL ≈ (5/3) × N × rollouts × L_total`（N=参数量, rollouts=6, L=2048）
- 分配比例 β ∈ [0,1]：`T_mid = (1-β)·T`，`T_RL = β·T`

**测试配置（相同 base model，相同总 compute）：**

| 配置 | β | 说明 |
|------|---|------|
| Full Mid-Training | ~0 | 1B supervised tokens，op=11–14 |
| Light-RL | 0.2 | 主要 mid-training |
| Medium-RL | 0.5 | 均等分配 |
| Heavy-RL | 0.8 | 主要 RL |
| Full RL | ~1.0 | 100 steps，batch=1024 |

**结论：**
- OOD-edge 任务：Full Mid-training + Light-RL 的 pass@1 **最优**
- OOD-hard 任务：Heavy-RL 在 pass@1 和 pass@128 均有显著提升
- Mid-training + RL vs. 纯 RL，固定 compute 下 OOD-hard 提升 **+10.8%**

**解释 Qwen vs. LLaMA 差异：** Qwen 对 RL 响应更好，很可能是因为 Qwen 已经做过 mid-training；LLaMA 在做 mid-training（OctoThinker，Wang et al.）后 RL 性能匹配 Qwen。

**实践指导（任务感知的 compute 分配）：**
- 追求 OOD-edge 可靠性 → 多 mid-training + 轻 RL
- 追求 OOD-hard 探索性 → 适量 mid-training 打基础 + 重 RL

### Finding 4：Process-level Reward 抑制 Reward Hacking（Section 6）

**Reward 设计对比：**

| 方案 | 公式 |
|------|------|
| 纯 outcome | `R = R_out`（答案对=1） |
| Process verification | `R = R_pv`（每步 DAG 节点准确率） |
| 加权混合 | `R = α·R_out + (1-α)·R_pv` |
| Strict | `R = R_out` 仅当 `R_pv = 1`，否则 0 |

**R_pv 的具体计算**：由于任务是 DAG 算术题，每个中间变量的正确值已知：
$$R_{pv} = \frac{\text{模型输出中计算正确的 DAG 节点数}}{\text{DAG 总节点数}}$$
这是连续值（非 0/1），等价于合成数据上的 perfect step-level PRM。

**最佳配置：**
- `0.2·R_out + 0.8·R_pv` → pass@1 提升 **+4–5%**（vs. 纯 outcome reward）
- Strict 方案效果更强

**机制**：纯 outcome reward 容易导致 reward hacking——模型通过 shortcut / pattern matching 碰对答案，跳过真实推理步骤。Process reward 权重高，强迫模型把中间步骤算对，抑制 shortcut。

**与真实场景的差距**：本文能做 process reward 是因为合成数据有 ground-truth DAG。真实 LLM/VLM 场景需要额外训练 PRM（Process Reward Model）才能实现类似效果。

## 对 ThothVL Mid-training 的启发

### 1. 数据质量/覆盖率 > 数据量/复杂度

Pre-training 中 context B 只需 1% 覆盖率就能解锁 RL 的泛化能力。MT 阶段值得在以下方向上花功夫：
- **Quality filtering**：确保数据的 structural correctness（而非 surface diversity）
- **Broad atomic coverage**：覆盖更多基础 skill types，而非堆叠复杂 multi-step 数据
- **Contamination-free split**：MT 数据应不与 RL/eval 重叠，避免虚高的 headroom

### 2. Mid-training 决定 RL 上界

本文定量证明了 mid-training + RL 在固定 compute 下显著优于纯 RL。意味着：
- ThothVL MT 阶段的质量直接决定后续 RL（RLVR/GRPO）能达到的天花板
- 如果 MT 数据覆盖不足，RL 只能做 sharpening，无法产生真实能力增益

### 3. RL 数据难度需对齐"能力边界"

- 可用 pass@1 ≈ 0 但 pass@k > 0 作为筛选条件，构建 self-paced RL curriculum
- 对 ThothVL：在视觉推理任务上，可用多次采样预筛选有效样本

### 4. Process Supervision 值得尝试

- 对有中间步骤注释的数据，使用 step-level reward 比纯 answer-level reward 更鲁棒
- 在 ThothVL 的 chain-of-thought 数据上，验证 intermediate steps 的正确性可能提升 RL 效果

---

## 局限性

1. **合成数据**：DAG 算术任务 ≠ 真实 VLM/LLM 任务，结论能否迁移到自然语言推理需验证
2. **小模型**（100M）：scaling 行为可能不同
3. **单一 RL 算法**（GRPO）：其他算法（PPO, DPO）可能表现不同
4. **单语言**：纯文本，无视觉模态
