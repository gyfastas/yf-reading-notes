01 — TL;DR 核心贡献

定位：首篇系统性综述 Mid-Training 领域的论文。提出了覆盖数据分布、学习率调度、长上下文扩展三大维度的统一分类体系。
核心论点：Mid-training 缓解了 noisy tokens 的递减收益，稳定了收敛过程，扩展了模型晚期训练的能力。其有效性可通过三个理论视角解释：
  • Gradient Noise Scale (GNS)：高质量数据提高梯度方差，帮助逃离 sharp minima
  • Information Bottleneck (IB)：高质量监督提供低熵指导，促进语义结构提取
  • Curriculum Learning：从多样/嘈杂语料逐渐过渡到挑战性样本，强化复杂推理

02 — Mid-Training 定义与定位

什么是 Mid-Training？

定义：Mid-training 是 Pre-training 后期到 Post-training 之前的中间阶段，通过在精选高质量数据上进行多阶段 annealing-style 训练，实现数据质量精炼、优化策略适配和上下文长度扩展。

核心目标：将模型从"大规模记忆"模式转向"高质量抽象"模式——当模型接近通用泛化的容量极限时，高质量样本通过引导 attention 聚焦于结构化推理、事实性和指令遵循来实现进一步增益。

在训练流程中的位置

阶段 数据规模 数据类型 学习率 目标
Pre-training 数万亿 tokens 通用互联网数据 Peak LR 通用能力、世界知识
Mid-training 数百亿 ~ 数千亿 tokens 精选领域数据 + 通用数据 Peak 的 10%–30% 领域知识注入、长上下文、质量精炼
Post-training 数十万 ~ 数百万条 指令 / 偏好数据 更低 对齐、指令遵循、RLHF
与 Continued Pre-training (CPT) 的区别：
  • CPT：直接在领域数据上继续训练，通常不改调度策略，容易遗忘
  • Mid-training：有意识地设计多阶段调度（数据混合 + LR decay + 上下文扩展），作为 pre→post 的"桥梁"

03 — 三维分类体系

Survey 的核心 Taxonomy：三个维度互相增强（mutually reinforcing），缺一不可。
  1. Data Distribution — 精选数据类型、混合比例、质量过滤
  2. Learning Rate Scheduling — Warmup / Decay / 多阶段策略
  3. Long Context Extension — RoPE 插值、分阶段扩展
没有精炼数据 → 缺乏专业化；没有合理调度 → 不稳定或浪费高质量 tokens；没有上下文扩展 → 受限于短序列。

04 — Learning Rate Scheduling 公式详解

关键发现：所有调度器共享结构 = Linear Warmup → Decay Phase，但 decay 形状影响收敛质量。Cosine decay 仍是经验性主流，但线性 decay 在某些设置下更优。Decay 策略的效果与 batch size、模型规模、优化器深度耦合，尚无统一理论。

4.1 Linear Scheduler

η_t = η_peak - (η_peak - η_end) · t / S_dc,   0 ≤ t ≤ S_dc

最简单的线性衰减。DeepSeek 系列使用 multi-step 变体（在固定 token ratio 处离散下降）。

4.2 Cosine Scheduler

η_t = η_end + 0.5 · (η_peak - η_end) · (1 + cos(π · t / S_dc))
关键设置：令 S_dc = S_total（decay 跨越整个训练长度）效果最佳——平衡了早期探索与后期收敛。LLaMA 3 用 cosine decay 到 peak 的 1%，再加 40M tokens 线性衰减到 0。

4.3 WSD Scheduler (Warmup-Stable-Decay)

WSD(T; s) = { (s / W) · η, s < W // Warmup η, W ≤ s ≤ T // Stable f(s - T) · η, T < s < S // Decay }
优势：三阶段设计，在最后 10% tokens 内实现快速 loss 下降。核心价值：Checkpoint 可复用——Stable 阶段的 checkpoint 可以直接接续不同的 decay 分支，适合多轮 mid-training。MiniCPM 系列采用此策略。

4.4 Power Scheduler

η_power(n) = min(η_peak, β · a · n^b) 三阶段：warmup → slow power-law decay → fast exponential decay

核心价值：支持零样本 LR 迁移——通过 power-law 关系在不同超参数间迁移最优 LR。

4.5 Multi-step Scheduler (DeepSeek 风格)

在固定 token 比例处（如 60%, 80%）离散降低 LR 例: η_peak → 0.316·η_peak → 0.1·η_peak

DeepSeek-V2/V3 采用。优势：简单可控，便于继续训练时复用中间 checkpoint。

4.6 Mid-training 阶段的 LR 最佳实践

参数 推荐范围 依据
Peak LR Pre-training peak 的 10%–30% 避免破坏已学知识
Warmup steps 较短（数百 ~ 2K steps） Pre-trained 权重已稳定
Decay 形状 Cosine 或 WSD Cosine 经验主流；WSD 支持 checkpoint 复用
Final LR Peak 的 1%–10% 过高影响收敛，过低浪费尾部 tokens
Batch size 可渐进增大 Larger batch → 支持 higher LR；减少梯度方差

5.3 Data Mixing 关键洞察

经验法则总结：
  1. STEM 数据优先：少量 Math/Code 数据可带来不成比例的推理增益
  2. 分布连续性防遗忘：OpenCoder 保留 84% mid-training 数据来自原始 pre-training 分布
  3. 指令数据时机未定：DeepSeek 在预训练末尾加 5M 指令样本取得与 SFT 相当的增益，但最优时机仍未定论
  4. 适度重复 > 缺乏数据：将任务重写为模型友好格式可显著提升性能
  5. 长上下文数据选择关键：自然长文档 >> 人工拼接文档
  6. 动态混合 > 静态混合：随训练阶段调整比例（如 SmolLM2）

5.4 Data Mixing Scaling Laws（相关工作）

论文 核心方法
Data Mixing Laws (ICLR 2025) 发现混合比例对性能的定量可预测性；嵌套使用 step / size / mixing 三层 scaling law 预测大模型性能
MergeMix (2601.17858) 线性权重插值作为数据梯度累积的高保真代理，将 data mix 优化转化为低成本模型合并问题
Capacity-Aware Mixture Law (2603.08022) Data mix scaling law 同时依赖混合比例和模型大小；MoE 7B-A150M 优化成本降低 50%，benchmark 提升 3%

06 — Long Context Extension (RoPE 插值详解)

统一公式：所有方法仅在频率重映射函数 h(θ) 上不同。
f'_W(x_m, m, θ_j) = f_W(x_m, m, h(θ_j)) = R_{h(θ_j), m} · W · x_m 原始 RoPE: θ_j = 10000^(-2(j-1)/d)

6.1 Position Interpolation (PI)

h_PI(θ_j) = α · θ_j, α = (L_train - 1) / (L_target - 1)
优点：概念简单，轻量微调，中等扩展倍率下鲁棒。
缺点：均匀压缩损伤高频（局部）信号——短距离位置区分度下降。

6.2 NTK-Aware Interpolation

h_NTK(θ_j) = (b')^(-2j/d), b' = b · s^(d/(d-2))
核心思想：低频扩展多，高频扩展少——保留局部位置分辨率的同时扩展远距离 attention。相比 PI，减少长程 attention decay。
问题：目标长度与曲率参数耦合，调参困难。

6.3 NTK-by-Parts（分段 NTK）

h(θ) = { θ // 高频：保持不变，保留局部细节 θ / s // 低频：完全压缩 θ / s^ρ(θ) // 中频：平滑插值 }

分段处理保留最短波长（局部细节），短上下文性能保持更好。

6.4 YaRN (Yet another RoPE extensioN)

在 NTK-by-Parts 基础上增加：长度依赖的 logit temperature t(s) 用于稳定化。

效果：以更少的数据/步数达到 32K–128K 上下文。快速扩展 + 稳健收敛的最佳平衡。

  • 多阶段渐进扩展

07 — 理论基础

7.1 Gradient Noise Scale (GNS)

原理：高质量数据产生更大的梯度方差（更高 GNS）。更高的 GNS 帮助模型逃离 sharp minima，避免在低质量数据的平坦区过拟合。

实践含义：Mid-training 切换到高质量数据 → GNS 升高 → 模型跳出 pre-training 末期的低收益 plateau → 进入新的下降通道。

7.2 Information Bottleneck (IB)

原理：表征学习本质是压缩——逐步减少对嘈杂/冗余特征的依赖。高质量监督提供更清晰的低熵指导，帮助模型：
  • 识别语义上有意义的结构
  • 衰减虚假相关
  • 实现从记忆到抽象的转变

7.3 Curriculum Learning

原理：LR annealing 天然与课程学习原则对齐——从多样/嘈杂语料逐渐过渡到挑战性/信息量大的样本。

效果：强化了多步推理和代码生成等复杂技能。SmolLM2 五阶段递进（Web → +Code → +Math → +Synthetic → +Long）正是 curriculum 的体现。

10 — 开放问题与研究前沿

尚未解决的关键问题：
  1. 指令数据最佳时机：Mid-training 中引入指令数据（如 DeepSeek 5M 样本）的效果不稳定，最优放置位置未定论
  2. Decay 策略理论缺失：Cosine vs Linear vs WSD 在不同系统中表现不一，缺乏统一理论解释
  3. Warmup 持续时间：缺乏理论指导，跨模型规模 / 优化器差异大
  4. Scheduler 敏感度矛盾：有证据表明"只要 LR 预算足够，形状不重要"，也有反例
  5. Checkpoint 复用 vs 最终性能：WSD 等方便复用，但可能牺牲最终 loss
  6. 过度预训练 (Overtraining)：过多 pre-training 反而降低后续 fine-tuning 适应性
  7. 多领域顺序 Mid-training：如何在多个领域间顺序训练避免前域遗忘？
  8. Mid-training 特有 Scaling Laws：不同于 Chinchilla，尚需专���研究

11 — 对 ThothVL Mid-Training 的启发

数据策略

  • 分阶段递进混合：参考 SmolLM2 模式——先巩固通用，逐步引入视觉推理 / OCR / Chart 等专门数据
  • 分布连续性 ≥80%：确保每阶段有足够比例的上一阶段数据重放
  • STEM 视觉数据优先：少量高质量 Math-Vision / Science-Diagram 数据可带来不成比例的增益
  • 结合 2512.07783 发现：至少 1% 的 atomic-level 覆盖率，确保后续 RL 能泛化

LR 策略

  • WSD 值得考虑：支持 checkpoint 复用，适合多轮实验迭代
  • Peak LR = Pre-train 的 10%–20%：VLM mid-training 对 LR 更敏感（视觉编码器权重更脆弱）
  • Cosine + 短 warmup：如果不需要 checkpoint 复用，cosine 仍是安全选择

长上下文

  • YaRN 或 NTK-by-Parts：保留短上下文视觉理解的同时扩展到长视频 / 多图场景
  • 渐进式扩展：参考 LLaMA 3 六阶段方案，避免一步到位的性能损伤
  • 必须混合短数据：短上下文视觉推理是核心能力，不能在长上下文训练中丢失

  • Mid-training 质量决定 RL 上界（Survey 的"桥梁论" + 2512.07783 的 Finding 3: mid-train + RL >> 纯 RL）
  • 数据广度 > 数据复杂度（Survey 的分布连续性 + 2512.07783 的 ≥1% 覆盖率阈值）
