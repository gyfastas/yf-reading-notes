Domain-Specific Mid-Training
领域持续预训练实践

医疗、法律、金融领域的中期训练案例总结

📌 概述

Mid-training的价值：在通用预训练模型基础上，通过领域特定的中期训练，可以在保持通用能力的同时获得专业的领域知识。
关键发现：
  • 领域模型在中期训练后显著超越通用模型
  • 数据质量和领域覆盖率比数量更重要
  • 需要平衡领域专业化和通用能力保持
  • 不同领域有不同的最佳实践

🏥 医疗领域

代表模型：PMC-LLaMA / HuatuoGPT / Meditron

数据：

  • PubMed Central (PMC) 全文文献
  • 医学教材和百科全书
  • 临床指南和诊疗规范
  • 医患对话数据（用于对齐）

规模：通常 50B-200B tokens 医学文本

最佳实践：

  • 混合10-20%通用数据防止遗忘
  • 学习率：Pre-training的10-15%
  • 重点：医学术语、疾病知识、诊疗逻辑

性能提升

基准 通用模型 医学Mid-training后
USMLE ~40% ~60%
PubMedQA ~50% ~75%
MedMCQA ~35% ~55%

⚖️ 法律领域

代表模型：SaulLM / Lawyer LLaMA / ChatLaw

数据：

  • 法律法规条文
  • 判决书和裁定书
  • 法律教材和论文
  • 法律问答数据

规模：通常 30B-100B tokens 法律文本

最佳实践：

  • 不同法域（民法/刑法/商法）分别处理
  • 重视判例推理链
  • 法条引用格式规范化

性能提升

基准 通用模型 法律Mid-training后
Legal Bench ~45% ~70%
Case Prediction ~50% ~75%

💰 金融领域

代表模型：BloombergGPT / FinGPT / XuanYuan

数据：

  • 财报和审计报告
  • 金融新闻和市场分析
  • 证券研究报告
  • 金融教材和论文

规模：通常 100B-500B tokens 金融文本

最佳实践：

  • 数值计算能力特别重要
  • 时序数据（股价、指标）处理
  • 多语言金融文本（中英文）
  • 实时数据更新机制

性能提升

基准 通用模型 金融Mid-training后
FinQA ~35% ~65%
Financial NER ~60% ~85%
Sentiment Analysis ~70% ~88%

🔍 跨领域共同模式

成功因素

1. 数据质量 > 数量
  • 精选高质量领域文本
  • 去重和清洗至关重要
  • 领域专家参与数据审核
2. 混合策略
  • 领域数据：70-80%
  • 通用数据：20-30%（防止遗忘）
  • 动态混合优于静态混合
3. 学习率配置
  • Peak LR：Pre-training的10-20%
  • Warmup：较短（5% steps）
  • Decay：Cosine decay到1% peak

常见陷阱

需要避免：
  1. 过度专业化：领域数据比例过高导致通用能力严重下降
  2. 数据污染：测试集数据混入训练集
  3. 领域过窄：只覆盖领域的一部分子领域
  4. 忽视对齐：Mid-training后需要专门的Post-training对齐

🛠️ 领域Mid-training实施建议

步骤清单：
  1. 需求分析：明确目标领域和评估基准
  2. 数据收集：构建高质量领域语料库
  3. 数据清洗：去重、过滤低质量内容
  4. 混合策略：确定领域/通用数据比例
  5. 超参配置：学习率、batch size、训练步数
  6. 监控训练：同时跟踪领域和通用能力
  7. Early Stopping：选择最佳checkpoint
  8. Post-training：指令微调和RL对齐
  9. 评估验证：全面评估性能和安全性
