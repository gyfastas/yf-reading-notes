01 — TL;DR

核心定位：首个万亿参数科学多模态基础模型。从 Intern-S1 (235B) 通过 4x 专家扩展到 1T/512 experts。覆盖化学、材料、生命科学、地球科学等 100+ 科学任务，同时保持 frontier 级通用能力（AIME 2025: 93.1）。
核心创新：
  • Grouped Router + STE：解决万亿 MoE 的 load balance 和 router 训练问题
  • Stable Mixed-Precision RL：FP8 expert + BF16 non-expert + FP32 LM head，万亿规模 RL 训练
  • Rollout Router Replay：RL rollout 和训练阶段保持 expert 选择一致性
  • FoPE + 时序编码器：Fourier Position Encoding + 自适应时序子采样
  • SAGE 框架：Foundation → Fusion → Evolution 三层科学能力构建

03 — Grouped Router + STE（核心创新）

问题

万亿 MoE 的两个挑战：
  1. Load Imbalance：512 experts 间负载极度不均衡，导致训练不稳定
  2. Router 优化困难：传统 Top-K sparse routing 中，未被选中的 expert 的 router embedding 收不到梯度

Grouped Router

方案：将 512 experts 均匀分成 8 组（对应 EP8 训练策略），每组内 Top-1 选择。

效果：
  • 绝对设备负载均衡：每个设备组恰好激活 1 个 expert，完全无负载倾斜
  • 最终激活集 = 8 组 x Top-1 = 8 experts per token
  • 比���统 Top-K 全局路由更稳定，尤其在超大 expert 数量下

STE (Straight-Through Estimator) Routing

前向：Sparse Top-K 选择（正常稀疏推理）
反向：Dense softmax 梯度流（所有 router embedding 都收到梯度）

意义：解耦了推理效率（sparse forward）和训练质量（dense backward），使 router 充分优化。

04 — 训练 Pipeline

Continued Pre-training

维度规模
总 tokens6T (image-text + text)
科学 image-text~270B tokens
Web image-text pairs~300M pairs (CC12M, LAION-COCO, SBU, LAION-2B, Wukong)
扩展方式Intern-S1 (235B) → 4x expert expansion → 1T

科学数据处理三策略

  1. Structured Scientific Data Transformation：模板构建 + 任务映射，将非结构化科学数据转为训练格式
  2. Scientific Data Diversification：Prompt 多样化 + Rollout 机制，增加数据变体
  3. System Prompt Isolation：科学数据与通用数据用独立 system prompt 隔离，避免冲突

Stable Mixed-Precision RL

万亿规模 RL 的精度方案：
组件精度原因
Expert 层FP8参数量最大，FP8 大幅节省显存/计算
Non-Expert 组件BF16Attention / Norm 等需要更高精度
LM HeadFP32低精度导致"分段水平线"（同 MiniMax CISPO 发现）

Rollout Router Replay

问题：RL 中 rollout 阶段（推理生成）和 training 阶段（梯度更新）的 expert 路由可能不一致，导致训练不稳定。

方案：Rollout 时记录每层选中的 expert 索引，训练时重放相同的路由决策，强制一致性。

其他 RL 技术

  • Dual Importance Sampling + Masking：过滤长尾异常
  • Leave-one-out Baseline：用于 advantage estimation
  • 训练框架：XTuner + LMDeploy，确保训练-推理精度严格一致
  • 效率：从 S1 到 S1-Pro (4x expert)，训练效率仅下降 ~20%

05 — SAGE 框架 (Synergistic Architecture for Generalizable Experts)

���层架构：
  1. Foundation：基础多模态能力（语言 + 视觉 + 时序）
  2. Fusion：跨模态科学数据融合（图像-文本-时序联合理解）
  3. Evolution：通过 RL 持续进化科学推理能力

FoPE (Fourier Position Encoding)

设计理念：将位置信息建模为 Fourier 级数，同时捕获离散 token 特性和连续波动特性。适合科学数据中频域信号的表示。

Adaptive Time-Series Encoder

自适应子采样：根据信号采样率动态决定 patch size 和 stride。处理范围：10^0 到 10^6 timesteps。

应用：光谱数据、传感器信号、地球科学时序等。

07 — 关键启发

  • Grouped Router 解决万亿 MoE 稳定性：8 组 x Top-1 实现绝对负载均衡，比全局 Top-8 更稳定
  • STE Routing 的实用价值：稀疏前向 + 稠密后向，让所有 router embedding 都充分训练
  • FP32 LM Head 是 MoE RL 的硬约束：Intern-S1-Pro 和 MiniMax CISPO 独立发现同一问题
  • Rollout Router Replay：MoE + RL 的独特问题——routing 不一致会破坏训练。解决方案简单但关键
  • Expert Expansion (4x) 效率：从 235B 到 1T 仅 ~20% 训练效率下降，说明 MoE 扩展的边际成本低
  • 科学数据隔离策略：System Prompt Isolation 避免科学数据与通用数据冲突，值得在 ThothVL 的领域数据处理中参考
